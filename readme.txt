    ps. Лучше ставить плагин через репозиторий, тогда плагин будет обновляться.

IDE автоматически обнаружит обновления себя и установленных плагинов и выдаст подсказки. Если для этого плагина есть обновление, вам будет предложено просмотреть журнал обновлений и выбрать, нужно ли обновлять его самостоятельно.
Щелкните меню IDE Проверить наличие обновлений ..., чтобы вручную проверить IDE и установленные обновления подключаемых модулей. Если для этого плагина есть обновление, вам будет предложено просмотреть журнал обновлений и выбрать, нужно ли обновлять его самостоятельно.
Для обновлений подключаемых модулей может потребоваться перезапуск среды IDE.


    Что касается платных триал плагинов. Вот несколько рекомендаций.

Информация о пробной версии платного плагина также будет сброшена.
Существует две версии плагина MyBatisCodeHelperPro, функции точно такие же, вы должны внимательно прочитать это при установке!
MyBatisCodeHelperPro (Marketplace Edition) можно сбросить!
MyBatisCodeHelperPro, сбросить нельзя!
Для некоторых платных плагинов (например, Iedis 2, MinBatis) вам может потребоваться удалить конфигурацию javaagent (если есть) и перезапустить IDE:
Если IDE не открывает проект, щелкните Welcome в интерфейсе приветствия: Configure-Edit Custom VM Options ...- Удалите строку, начинающуюся с -javaagent :.
Если среда IDE открывает проект, щелкните меню: Help->Edit Custom VM Options... - Удалите строку, начинающуюся с -javaagent.
Чтобы сбросить настройки, необходимо перезапустить среду IDE!
После сброса диалоговое окно «Licenses» не появляется, чтобы вы могли выбрать лицензию или опробовать ее, что отличается от предыдущего сценария / плагина сброса (сохраните этот раздражающий шаг).
Если в течение 25 дней не было никаких действий по сбросу, IDE уведомит вас, чтобы спросить, хотите ли вы выполнить сброс.
Если вы отметите: Auto reset before per restart, сброс будет тихим и незаметным.
Проще говоря: поставьте Auto reset before per restart, и не нужно беспокоиться об этом раз и навсегда.
